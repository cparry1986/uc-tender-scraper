Usage Guidelines

Brand Voice & Tone

Modern & Technical: We speak the language of energy, blockchain, and sustainability without being overly technical.

Optimistic & Empowering: We inspire users to take control of their energy future.

Transparent & Honest: We clearly explain how our platform works and its benefits.

Local & Community-Focused: We emphasize peer-to-peer, local, and community aspects.

Color Usage Guidelines

  • Primary Cyan (#00E5BF): Use for primary CTAs, active states, highlights, and main interactive elements.
  • Secondary Purple (#6C5CE7): Use for secondary actions, accent cards, and alternative CTAs.
  • Accent Lime (#D4FF00): Use sparingly for badges, decorative elements, and energy-themed highlights.
  • Navy (#0D0B2E): Dark backgrounds for hero sections and dark cards. Use white text.
  • White & Grays: Use for text, backgrounds, and neutral elements. Maintain sufficient contrast.

Typography Guidelines

  • Use Display (64px) for page hero statements.
  • Use H1 (48px) for main page titles.
  • Use H2-H3 (36px-28px) for section headings.
  • Use Body text (16px) for standard paragraph content.
  • Maintain 1.6 line-height for body text for readability.
  • Use weights: 400 for body, 600 for headings, 700 for display.

Component Best Practices

  • Buttons: Primary cyan for main CTAs. Secondary for alternatives. Pill shape (border-radius: 24px) for prominent actions.
  • Cards: Light cards for content. Dark hero cards with diagonal accents for feature announcements.
  • Forms: Clear labels, visible focus states (cyan border + light background). Required field indicators with red asterisk.
  • Spacing: Maintain consistent spacing using the 4px unit system.
  • Shadows: Use md-lg shadows for depth. Avoid excessive shadows.

Icon Recommendations

For icons, we recommend using:

  • Lucide React: Modern, minimal icon set that matches our aesthetic
  • Phosphor Icons: Geometric, clean icons available in multiple weights

Size icons at 24px (standard), 16px (small), 32px (large). Color icons with brand colors or inherit text color.

Dark Mode Support

All components support dark mode with appropriate color adjustments. Use CSS variables with dark-mode class for consistent theming.

Test all components in both light and dark modes. Ensure sufficient contrast ratios (WCAG AA or better).

DO's and DON'Ts

DO

  • Use cyan (#00E5BF) as primary color
  • Maintain consistent spacing (4px unit)
  • Use DM Sans font family
  • Include focus states on interactive elements
  • Support keyboard navigation
  • Test accessibility (WCAG)
  • Use semantic HTML

DON'T

  • Mix brand colors arbitrarily
  • Use inconsistent spacing
  • Overuse lime accent color
  • Remove focus indicators
  • Use poor color contrast
  • Create custom button styles
  • Ignore responsive design
